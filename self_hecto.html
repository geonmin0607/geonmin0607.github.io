(주)헥토 AX SEED팀 AI 엔지니어로 지원합니다. 오픈소스·허깅페이스 모델을 기반으로 한 SFT/PEFT/DPO 실험과, 문서·이미지 도메인 데이터 전처리·파이프라인 설계 경험을 보유하고 있습니다.

특히 PyTorch + Hugging Face 스택에서 Qwen·DeepSeek 계열 모델을 활용해 태스크를 정의하고, FastAPI 기반 API까지 연결해 내부 사용자가 바로 쓸 수 있는 형태의 도구를 만들어 왔습니다.

1. 지원 동기

저는 모델 성능만 올리는 것보다, 데이터를 설계하고 학습·추론 구조를 만들어 업무 현장에서 바로 쓸 수 있는 AI 도구를 만드는 일에 가장 큰 보람을 느낍니다. 현재 회사에서는 검색·규정/법령·건축민원·문서 분석 등 여러 도메인에서 LLM·VLM을 활용한 서비스를 기획하고, 데이터·모델·API 구조를 설계하는 역할을 맡아 왔습니다.

헥토 AX SEED팀 공고에서 “오픈소스 및 허깅페이스 모델 리서치 및 서비스화”, “Pretrain, SFT, PEFT, RLHF, DPO 등 다양한 방식의 모델 개발”을 요구하는 것을 보고 지금까지 제가 집중해 온 방향과 맞닿아 있다고 느꼈습니다. 저는 문서·이미지 도메인 중심으로 SFT/PEFT·DPO를 실제 프로젝트에 적용해 보았고, 아직 경험하지 못한 Pretraining·본격적인 RLHF·클라우드 환경 워크플로우는 앞으로 확장하고 싶은 영역입니다.

헥토에서는 모빌리티·금융·헬스케어·라이프 등 여러 서비스에서 발생하는 데이터를 기반으로, Hugging Face 생태계의 모델들을 빠르게 검증하고, 실제 서비스에 녹일 수 있는 실행력 있는 AI 엔지니어로 기여하고자 합니다.

2. 직무 관련 경험

1) 데이터 전처리 및 검색/QA 모델링

지능형 검색 서비스 프로젝트에서는 약 380개 웹사이트를 대상으로 크롤러·스크래퍼를 설계하고, 수집→정제→색인 파이프라인을 구축했습니다. Airflow로 배치를 자동화하고, 오류율·지연·처리량 지표를 두어 수집 실패와 사이트 구조 변경을 조기에 발견했습니다. Elasticsearch 기반으로 키워드와 벡터 검색을 결합해 사용자가 표현을 다르게 해도 유사 문서를 찾을 수 있도록 랭킹을 구성했습니다.

2) LLM 학습 및 평가 구조 설계

발전사 업무 지원 프로젝트에서는 규정·법령·매뉴얼을 AI가 읽기 좋은 단위로 청킹하고, 조항/개정일/적용대상 등의 메타데이터 스키마를 설계해 RAG 검색 품질을 높였습니다. 건축민원 프로젝트에서는 민원 데이터를 분석해 질문 유형을 정의하고, 복수 질문을 자동 분리하는 프롬프트와 파이프라인을 설계했습니다. 이후 LLM-as-a-Judge 기반 자동 평가와 오류 패턴 분석을 통해 데이터·프롬프트·모델 중 어디를 수정해야 효과적인지 반복적으로 실험했습니다.

멀티모달 PoC에서는 Qwen2.5-VL, Qwen3-VL, DeepSeek-OCR 등 허깅페이스 기반 모델에 LoRA·QLoRA를 적용해 표·차트·도식·화학 구조를 HTML·코드·SMILES로 변환하는 태스크를 학습했습니다. 이 과정에서 해상도별 커리큘럼 학습, loss 마스킹, Vision Prior 설계를 적용해 구조적 오류와 환각을 줄이는 실험을 진행했습니다.

3) API 설계 및 실험 환경

모델을 실제로 활용하기 위해 FastAPI 기반 태스크별 엔드포인트를 설계했습니다. 예를 들어 “표→HTML”, “차트→Mermaid”, “문서→캡션”과 같이 기능별로 분리된 API를 구성하고, 상위 오케스트레이터가 페이지 단위 요청을 각 엔드포인트로 라우팅해 결과를 합치는 구조를 설계했습니다. vLLM과 Transformers 환경에서 동일 모델을 비교하며 지연·메모리 사용량을 측정했고, ClearML로 실험 결과와 로그를 관리했습니다. 아직 대규모 프로덕션 트래픽을 직접 운영한 경험은 없지만, API 설계와 실험 환경 구성, 성능 관찰·분석에는 익숙합니다.

3. 일하는 방식과 성향

저의 강점은 문제를 감으로 판단하지 않고 데이터와 로그를 기준으로 구조화해 보는 습관입니다. 수학 전공을 통해 벡터공간과 선형성에 대한 감각을 쌓았고, 이를 업무에서는 “현상을 변수와 지표로 표현해 보는 것”으로 확장해 왔습니다. 수집 실패나 모델 품질 저하가 발생하면, 데이터 분포 변화, 프롬프트/하이퍼파라미터, 추론 경로, 사용자 입력 패턴 등 여러 가설을 세워 로그와 샘플을 분석하며 원인을 좁혀 갑니다.

또 하나의 특징은 작은 단위의 실험을 빠르게 돌린 뒤, 결과를 문서와 코드로 남겨 팀과 공유하는 방식입니다. “이 설정이 왜 효과가 있었는지/없었는지”를 정리해 두면, 이후 비슷한 문제를 만났을 때 팀 전체의 탐색 비용을 줄일 수 있다고 생각합니다. 반대로 한 번 몰입하면 범위를 넓히려는 경향이 있어, 최근에는 프로젝트 초반에 필수 목표와 마감 기준을 명확히 정하고 1차 릴리스 전까지는 그 범위 안에서만 움직이도록 스스로를 관리하고 있습니다.

헥토 AX SEED팀에 합류하게 된다면, 제가 가진 오픈소스 모델 실험 경험과 데이터·프롬프트·평가 설계 역량을 기반으로 허깅페이스 모델의 SFT/PEFT·DPO 실험을 주도하고, 아직 경험하지 못한 Pretraining·RLHF·클라우드 기반 배포 영역까지 책임감을 가지고 빠르게 학습해 나가겠습니다.
